# Telegram VPN Bot for 3x-ui

Telegram-бот для автоматической выдачи VPN-доступа через панель [3x-ui](https://github.com/MHSanaei/3x-ui). Пользователь отправляет `/start`, бот создаёт клиента во всех настроенных inbound'ах и возвращает ссылки подключения с QR-кодами.

## Возможности

- Автоматическое создание клиентов в 3x-ui
- Идемпотентность: повторный `/start` не создаёт дубли
- Поддержка нескольких inbound'ов
- Subscription-ссылка с единым `subId`
- QR-коды для подключения
- Контроль доступа через allowlist Telegram ID

## Требования

- Python 3.10+
- Доступ к панели 3x-ui (API должен быть доступен по HTTP)
- Telegram Bot Token (получить у [@BotFather](https://t.me/BotFather))

## Установка

```bash
git clone <repo-url>
cd vpn-from-kirill-telegram-bot
make vendor        # создаёт venv и ставит зависимости
cp .env.example .env
# Отредактируйте .env — заполните все переменные
```

## Настройка .env

| Переменная             | Обязательная         | Описание                                                  |
| ---------------------- | -------------------- | --------------------------------------------------------- |
| `TELEGRAM_BOT_TOKEN`   | Да                   | Токен бота от BotFather                                   |
| `XUI_HOST`             | Да                   | Хост панели 3x-ui (домен или IP)                          |
| `XUI_PORT`             | Нет (по умолч. 2053) | Порт панели                                               |
| `XUI_WEBBASEPATH`      | Нет (по умолч. `/`)  | Базовый путь панели                                       |
| `XUI_USERNAME`         | Да                   | Логин администратора 3x-ui                                |
| `XUI_PASSWORD`         | Да                   | Пароль администратора 3x-ui                               |
| `XUI_INBOUND_IDS`      | Да                   | ID inbound'ов через запятую (например `1,3`)              |
| `ALLOWED_TELEGRAM_IDS` | Нет                  | Telegram ID через запятую, `*` для всех, пусто = никто    |
| `BLOCKED_TELEGRAM_IDS` | Нет                  | Telegram ID через запятую, `*` для всех, пусто = никто    |
| `SUB_URL_BASE`         | Нет                  | Базовый URL подписки (например `https://server:2096/sub`) |

### Как найти ID inbound'ов

1. Зайдите в панель 3x-ui
2. Перейдите в раздел **Inbounds**
3. ID отображается в первом столбце таблицы

## Subscription URL (подписка)

Бот поддерживает subscription-ссылки через настройку `SUB_URL_BASE`. При создании клиента бот задаёт одинаковый `subId` во всех inbound'ах, что позволяет использовать единую ссылку подписки.

**Важные нюансы:**

- Subscription endpoint должен быть включён в настройках 3x-ui (Panel Settings → Subscription)
- URL подписки может работать на другом порту (например 2096) — это зависит от настроек панели
- Формат `SUB_URL_BASE`: полный URL без trailing slash (например `https://your.server.com:2096/sub`)
- Итоговая ссылка: `{SUB_URL_BASE}/{telegram_user_id}`
- Если `SUB_URL_BASE` не задан, бот отправит только прямые ссылки подключения

**Как проверить, что подписка работает:**

1. Задайте `SUB_URL_BASE` в `.env`
2. Отправьте `/start` боту
3. Откройте subscription-ссылку в браузере — должен вернуться base64-список конфигов

Если ссылка не работает — проверьте настройки Subscription в панели 3x-ui.

## Ограничения

- Поддерживается только протокол VLESS (остальные протоколы пропускаются с предупреждением в логах)
- Бот не управляет inbound'ами — только создаёт клиентов
- Нет удаления/отключения клиентов через бота
- Один сервер 3x-ui
- Нет собственной базы данных — 3x-ui единственный источник истины
